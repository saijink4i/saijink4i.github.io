### 1.要件

最近、PC全体の権限をAIエージェントに与えて操作させる「OpenClaw」などのツールが話題になっています。しかし、メイン機のリモート操作をAIに任せるのはセキュリティ的に不安が残ります。
その対策として、AIエージェント実行専用に「Mac mini」を購入する人が続出し、売り切れ騒動まで起きたというニュースを目にしました。

そこで私は「手元で眠っているRaspberry Pi 4を活用できないか？」と考えました。性能は最新PCに劣るためOpenClawなどの活用はできないが、完全に分離された環境であり、かつローカルLLMを活用することで、機密性の高いコードも外部に漏らさずにAI開発ができる「自分専用クラウド」を構築することにしました。

### 2. 構築環境

今回の構築では、携帯やiPadからいつでもアクセスでき、かつセキュリティを担保するために以下の構成を採用しました。

- ハードウェア: Raspberry Pi 4 (8GB RAM)
- OS: Ubuntu 24.04 LTS
- IDE: code-server (VS Codeをブラウザで動かす環境)
- ネットワーク: Tailscale (複雑な設定不要のメッシュVPN)
- AIエンジン: Ollama (ローカルLLM実行) + Continue (VS Code拡張機能)

### 3. 環境構築手順

#### 3.1 システム最新化

```plain
sudo apt update && sudo apt upgrade -y
```

#### 3.2 Tailscaleの導入 (VPN・リモートアクセス)

```plain
curl -fsSL https://tailscale.com/install.sh | sh
```

認証と起動を行います。

```plain
sudo tailscale up
```

表示されたURLにアクセスしてログインし、認証を完了。
自動起動の設定します。

```plain
sudo systemctl enable --now tailscaled
```

#### 3.3 code-serverの構築 (Web IDE)

```plain
curl -fsSL https://code-server.dev/install.sh | sh
```

外部アクセス設定を行います。

```plain
# 設定ファイルを開く
nano ~/.config/code-server/config.yaml
```

`bind-addr: 0.0.0.0:8080` に変更します。
`password:` の後の文字列をメモ（ログインに使用）

サービス有効化です。

```plain
sudo systemctl enable --now code-server@$USER
```

#### 3.4 Ollamaの導入 (ローカルAIエンジン)

```plain
curl -fsSL https://ollama.com/install.sh | sh
```

コーディングモデルのダウンロードします。
ラズパイ4（8GB）でも動ける軽量モデルを選択します。

```plain
ollama run deepseek-coder:1.3b
```

#### 3.5 Continue拡張機能の連携

ブラウザで`http://[ラズパイのTailscale IP]:8080`にアクセスします。
拡張機能マーケットプレイスから **「Continue」** をインストールします。
**config.jsonの設定:** Continueのアイコンをクリックし、下部の設定アイコンから `config.json` を開き、以下を追記します。

```plain
{
  "models": [
    {
      "title": "Ollama (Local)",
      "provider": "ollama",
      "model": "deepseek-coder:1.3b"
    }
  ],
  "tabAutocompleteModel": {
    "title": "Ollama Autocomplete",
    "provider": "ollama",
    "model": "deepseek-coder:1.3b"
  }
}
```
